==========================================
具体的な仕様詳細
==========================================

【データ構造】
public/data/kanji-grade5.json の okurigana 問題を確認:
- kanji フィールド(例: "喜ぶ")
- reading フィールド(例: "よろこぶ")
これだけでは漢字部分と仮名部分の区切りが自明でないので、
データ構造を拡張するか、kanji と reading から自動推定する。

【自動推定の方法(おすすめ)】
1. kanji 文字列から、ひらがな部分と漢字部分を分離
   "喜ぶ" → 漢字部分:"喜"(1文字), 仮名部分:"ぶ"(1文字)
   "確かめる" → 漢字部分:"確"(1文字), 仮名部分:"かめる"(3文字)
   "明るい" → 漢字部分:"明"(1文字), 仮名部分:"るい"(2文字)
   "考え方" → これは難しい(2つに漢字が分かれる)が、5年生では稀なので無視

2. reading 文字列から、漢字部分の読みと仮名部分の読みを分離
   "よろこぶ" を kanji の "喜ぶ" に対応:
     漢字部分の読み: "よろこ"(reading の最後 1文字を除いた部分)
     仮名部分の読み: "ぶ"(reading の最後 1文字)
   ※ 仮名部分の文字数だけ末尾から取り出す
   "たしかめる" を kanji の "確かめる" に対応:
     漢字部分の読み: "たし"(reading の最後 3文字を除いた部分)
     仮名部分の読み: "かめる"(reading の最後 3文字)

3. もしこの推定が困難なケース(漢字が2つ以上ある送り仮名問題など)が
   あれば、そういう問題は jukugo や single にタイプを変更するか、
   個別対応する(レポートで報告)

【UIの実装】
- 大マス(漢字用): 既存の DrawingCanvas をそのまま使う
- 小マス(仮名用): 既存の DrawingCanvas を縮小サイズ(60〜70%)で使う
- 横一列に並べて、各マスの下に読み仮名ラベル
- マス1つにつき1文字を書く前提
  - "喜ぶ" → 大マス1個 + 小マス1個
  - "確かめる" → 大マス1個 + 小マス3個
  
【認識処理】
- 大マスには KanjiCanvas の漢字認識
- 小マスには KanjiCanvas のひらがな認識
- 採点時、各マスの認識結果を連結して kanji フィールドと比較

【仮名間違いの扱い】
- 漢字は正解だが送り仮名を間違えた場合(例「喜こぶ」と書いた)も「不正解」
  → 採点画面で「正解は『喜ぶ』(よろこぶ)」と表示することで学習機会に
- これでよければ追加の特別処理は不要
