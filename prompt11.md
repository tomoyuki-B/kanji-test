小学6年生の漢字データ(191字)を整備してください。
これは子どもが実際に使うアプリのデータなので、正確性が最重要です。

==========================================
1. 配当漢字リストの確定
==========================================
文部科学省「小学校学習指導要領(平成29年告示)」現行版の小学6年生
配当漢字 全191字 を採用します。

【信頼できる出典の例】
- 文部科学省 学習指導要領
- 学習指導要領LOD
- Wikipedia「学年別漢字配当表」(現行版を参照、平成元年版は不可)

【特に注意点】
平成29年告示で6年生から外れた字: 株、巻、看、簡、難、否、認、納、班、秘、棒、模、欲、訳、覧、臨、朗、論
※これらは6年生に含めないでください
平成29年告示で6年生に追加された字: 我、敬、系、絹、権、憲、源、厳、皇、鋼、刻、穀、骨、困、砂、座、済、裁、策、宗、尊、揮、預 など

【絶対NG】
旧版(平成元年版)の6年生リストを使うこと。

==========================================
2. 各漢字の問題データ作成
==========================================

【1問あたりのデータ形式】(既存5年生JSONと同じ)
{
  "id": "g6-001",
  "grade": 6,
  "unit": "",   // 6年生の単元情報は別タスクで対応するため空文字でOK
  "kanji": "...",
  "reading": "...",
  "type": "jukugo" | "okurigana" | "single",
  "example": "...",
  "meaning": "..."
}

【問題タイプの配分】
熟語:送り仮名:単漢字 = 4:3:3 程度を目安に。
漢字によって自然な形を優先。

【1漢字あたりの問題数】
原則1問。よく使う漢字は2問でもOK。
総数は190〜220問程度を目安に。

【例文の制約 - 最重要】
example の中に使ってよい漢字は以下のみ:
- 1〜6年生に配当された漢字(計1026字)
- 解答する漢字そのもの
中学校以降の漢字や常用漢字外は使わない。
迷ったらひらがなにする。

【意味の説明】
6年生が読んで理解できる平易な日本語で1〜2文。

【既存5年生問題との整合性】
- public/data/kanji-grade5.json は変更しない
- IDの重複に注意(5年生は g5-XXX、6年生は g6-XXX)

==========================================
3. 自己チェック(必須)
==========================================
データ作成後、以下を自分で必ず実施:

(a) 漢字数チェック: 191字すべて含まれているか、抜けや重複がないか
(b) 配当チェック: 6年生から外れた字(株、巻、看、簡、難、否、認、納、
   班、秘、棒、模、欲、訳、覧、臨、朗、論など)が混じっていないか
(c) 例文チェック: 各 example が1〜6年配当漢字 + 解答漢字 +
   ひらがな・カタカナのみで構成されているか
(d) 読みチェック: kanji と reading が常用漢字表の音訓と整合するか
(e) ID チェック: g6-001 から連番になっているか

==========================================
4. ファイルの出力
==========================================
- 出力先: public/data/kanji-grade6.json (上書き)
- 既存ファイル(サンプル数問)は完全に置き換えてOK
- インデント2スペースで読みやすく整形

==========================================
5. レポート
==========================================
完成後、以下を報告:
- 確定した191字の確認(あれば抜けや余り)
- 問題タイプの内訳(熟語○問、送り仮名○問、単漢字○問)
- 自己チェックで発見した問題と修正内容
- 完成した問題総数

【単元情報について】
6年生の教科書情報はまだ提供されていないので、unit フィールドは
空文字 "" のままで構いません。後日、教科書情報を提供したときに
別タスクで紐付けします。

子どもの教育に使うデータなので、時間がかかっても正確性を優先してください。
