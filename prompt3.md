ホーム画面(HomePage.tsx)を作り込んでください。

【画面の役割】
- アプリを開いて最初に表示される画面
- お子さん(小学校5〜6年生)がテスト開始までの設定をする
- 学習モチベーションのため、サマリー情報も表示する

【画面構成】 上から順に:

1. ヘッダー
   - アプリタイトル「漢字テスト」
   - シンプルで子どもが親しみやすいデザイン

2. サマリーカード(3つ横並び)
   - 「今日の正答率」(%)
   - 「連続学習日数」(日)
   - 「といた問題数」(累計、問)
   - データはまだ無いので、IndexedDBから取得を試みて、無ければ「-」表示でOK
   - 数字は大きめ(28〜32px)、ラベルは小さめ

3. モード選択(2択、大きめのカード)
   - 「書き取り」 アイコンつき
   - 「読み」 アイコンつき
   - 選択中のものはハイライト表示
   - 初期値は「書き取り」

4. 学年選択(2択)
   - 「5年生」「6年生」のボタン
   - 初期値は「5年生」

5. 出題範囲選択
   - 以下から1つを選択(ラジオボタン形式または選択カード)
   - 「ランダム」 全範囲からランダム出題
   - 「単元別」 → 選ぶと、選んだ学年の単元一覧が下に展開される(public/data の JSON の unit フィールドから取得)
   - 「苦手な漢字のみ」 IndexedDBに苦手データが無い場合は選択不可(グレーアウト)
   - 初期値は「ランダム」

6. 出題数選択(3択)
   - 「5問」「10問」「20問」のボタン
   - 初期値は「10問」

7. 「テスト開始」ボタン
   - 画面下部に大きく配置
   - 紫色(#534AB7)で目立たせる
   - タップするとモードに応じて /quiz/writing または /quiz/reading に遷移
   - 選択した条件は遷移先にState/queryParamsで渡す

【デザイン】
- 全体的にやさしい雰囲気で、子どもが親しみやすく
- ボタンや選択カードは指でタップしやすく(最低 48px 高さ)
- 配色は紫(#534AB7)を主に、背景は薄い水色や白
- フォントは丸みのあるものを優先(Tailwindのfont-sansのままでOK)
- iPad縦持ち・横持ちどちらでも見やすく

【データ取得】
- src/lib/db.ts のIndexedDBラッパーを使ってサマリー情報を取得
- 関数が未実装なら、HomePageから呼び出す前提のインターフェースだけ追加してOK
- 必要なら getStats() のような関数を db.ts に追加してください

【遷移先に渡す情報】
- mode: "writing" | "reading"
- grade: 5 | 6
- range: { type: "random" | "unit" | "weak", unit?: string }
- questionCount: 5 | 10 | 20

修正後、変更したファイルと、ホーム画面の機能を箇条書きで教えてください。
