テストの流れ全体を完成させてください。
具体的には、複数問進行・採点画面・結果画面・成績保存を実装します。

【全体の流れ】
ホーム画面 → 出題画面(1問目→2問目→...→N問目) → 結果画面 → ホーム画面に戻る
各問の解答後に採点画面を挟むかどうかは下の仕様参照。

==========================================
1. 複数問進行
==========================================
【現状】 currentIndex が常に 0 のまま、次の問題に進めない
【修正】
- 出題画面 (QuizWritingPage / QuizReadingPage) で現在の問題インデックスを正しく管理
- 「こたえる」ボタン押下時の流れ:
  1. 解答を採点(後述)
  2. 採点画面を表示(0.5〜1秒の正誤フィードバック)
  3. 自動で次の問題へ進む。または「次へ」ボタンで進む
  4. 全問終了したら結果画面に遷移
- 各問の解答結果(正誤、ユーザーの解答、正解、問題ID)を配列で保持
- 進捗表示(「3/10」)を currentIndex に応じて更新

【採点画面の表示方式】
2つの選択肢のうち、後者(B)で実装してください:
- A: 別画面に遷移 → ボタンで次へ(画面遷移が多くてうるさい)
- B: 出題画面の上に重ねるオーバーレイで「○正解!」「×不正解…」を1.5秒表示 → 自動で次へ
  - 正解: 緑のチェックマーク + 「正解!」 + 問題の読みも表示
  - 不正解: 赤の×マーク + 「正解は『興奮』(こうふん)」を表示
  - 効果音は今は無し(後で追加可能)

3. 結果画面 (ResultPage.tsx) の実装
==========================================
全問終了時に表示する画面。
【表示内容】
- 「テスト終了!」のような大きな見出し
- 正答率(例: 80% / 10問中8問正解)
- 大きな正答率の円グラフまたは数字
- 間違えた問題の一覧:
  - 問題の漢字または読み(問題のタイプによる)
  - ユーザーの解答
  - 正解
  - 例文
- 「もう一度」ボタン(同じ条件でテスト再開)
- 「ホームへもどる」ボタン

【遷移時のデータ受け渡し】
- 出題画面から useNavigate の state で結果データを渡す
- もしくは Context APIで管理する

==========================================
4. 成績保存(IndexedDB)
==========================================
【保存タイミング】
- 各問の解答時点、または結果画面遷移時にまとめて保存
【保存内容】
- 各問の解答記録: { questionId, kanji, reading, userAnswer, isCorrect, mode("writing"|"reading"), timestamp }
- セッション記録: { sessionId, mode, grade, range, questionCount, correctCount, startedAt, finishedAt }

【lib/db.ts の関数を整備】
- saveAnswer(record): 1問の解答を保存
- saveSession(record): 1セッションを保存
- getRecentSessions(limit): 直近のセッション一覧を取得
- getStats(): { todayAccuracy, streakDays, totalAnswered } を返す
- getWeakKanji(limit): 間違えた回数の多い問題を集計して返す
   返り値: { questionId, kanji, reading, mistakeCount }[]
